# AI Summary 项目功能测试计划

## 一、项目功能概述

### 1.1 核心功能
AI批量文本总结工具，使用AI模型批量处理txt文件并生成markdown格式的总结。

### 1.2 功能模块

| 模块 | 功能点 | 描述 |
|------|--------|------|
| **AI提供商管理** | 添加服务商 | 新增AI服务商（名称、base_url、api_key） |
| | 编辑服务商 | 修改服务商配置 |
 | 删除服务商 | 软删除到回收站 |
 | 添加模型 | 为服务商添加模型 |
 | 删除模型 | 从服务商删除模型 |
| **提示词管理** | 添加Prompt | 新增自定义提示词 |
 | 选择Prompt | 选择使用的提示词模板 |
 | 查看内容 | 查看当前Prompt详细内容 |
 | 删除Prompt | 软删除到回收站 |
| **文件处理** | 扫描文件 | 扫描目录下的txt文件 |
 | 批量处理 | 调用AI API处理文件 |
 | 保存结果 | 生成md文件 |
 | 进度显示 | 实时显示处理进度 |
 | 取消处理 | 中途取消处理任务 |
| **回收站** | 恢复 | 从回收站恢复提供商/Prompt |
 | 永久删除 |彻底删除 |
| **用户偏好** | 自动保存 | 记住用户的选择 |
 | 恢复偏好 | 下次打开自动恢复 |

## 二、测试用例设计

### 2.1 配置管理测试

#### TC-001: 验证配置加载
- **操作**: 启动应用，访问首页
- **预期**: 正确加载config.json中的提供商和Prompt配置
- **验证点**: 下拉菜单显示正确的选项

#### TC-002: 添加新服务商
- **操作**: 通过UI添加新的AI服务商
- **输入**:
  - 服务商名称: "TestProvider_UnitTest"
  - Base URL: "https://api.test.com"
  - API Key: "sk-test-key-123"
  - 模型: {"TestModel": "test-model-id"}
- **预期**: 服务商成功保存并显示在下拉菜单中
- **验证点**: 刷新页面后新服务商仍然存在

#### TC-003: 添加新Prompt
- **操作**: 通过UI添加新的提示词
- **输入**:
  - Prompt名称: "测试Prompt_UnitTest"
  - Prompt内容: "你是一个测试助手"
- **预期**: Prompt成功保存并显示在下拉菜单中

#### TC-004: 删除服务商到回收站
- **操作**: 删除刚才添加的测试服务商

- **验证点**: 回收站显示该服务商，有恢复按钮

#### TC-005: 恢复服务商
- **操作**: 从回收站恢复服务商
- **预期**: 服务商恢复到列表中
- **验证点**: 刷新后服务商重新出现在下拉菜单

#### TC-006: 永久删除
- **操作**: 永久删除回收站中的服务商
- **预期**: 服务商彻底消失
- **验证点**: 回收站中也不再显示
